# Circle Battle

一个最小可运行的 2D 联机圆点大战小游戏，使用 HTML Canvas、Node.js、Express 和 Socket.IO。

## 运行方式

```bash
npm install
npm start
```

然后在浏览器打开：

```text
http://localhost:3000
```

可以打开多个浏览器窗口测试多玩家联机。

## 操作

- 进入网页后输入玩家 ID，并选择颜色
- 右键点击地图：移动
- 左键点击地图：射击
- 左下角按钮可以切换模式和地图
- 左下角按钮可以开启/关闭地图道具
- 1/2/3/4：切换手枪、冲锋枪、霰弹枪、狙击枪
- R：换弹
- 右侧状态栏可以查看自己和对手的血量、K/D、武器、弹药、换弹和增益状态
- 玩家血条旁会显示小型增益图标，方便在地图上观察加速和护盾状态

服务器负责玩家移动、子弹生成、碰撞、扣血、死亡复活和状态广播；客户端只负责输入和渲染。

服务器默认以 60 tick/s 更新和广播状态，让移动和射击反馈更顺滑。

## 游戏模式

- 死亡竞技：死亡后 3 秒自动复活，K/D 实时累计。
- 单场生存：一局开始后死亡不复活，直到只剩最后一名玩家；决出胜者后所有人复活并自动开始下一局，K/D 继续累计。

## 武器

- 手枪：12 发弹夹，稳定单发，标准移动速度。
- 冲锋枪：25 发弹夹，低伤害高射速，较轻，移动更快。
- 霰弹枪：5 发弹夹，一次发射 6 颗散射弹丸，较重，移动略慢。
- 狙击枪：4 发弹夹，高伤害低射速，最重，移动最慢。

武器、弹夹、冷却和换弹全部由服务器判定，客户端只负责发送切枪、换弹和射击方向。

武器平衡数据放在：

```text
config/weapons.json
```

常用字段：

- `magazineSize`：弹夹容量
- `damage`：单颗子弹伤害
- `bulletRadius`：子弹半径
- `bulletSpeed`：按 30 tick/s 手感填写的基础速度
- `bulletLifeSeconds`：子弹存活秒数
- `cooldownMs`：射击冷却毫秒数
- `reloadMs`：换弹毫秒数
- `pelletCount`：一次射击发射的弹丸数
- `spreadDegrees`：散射角度
- `moveSpeedMultiplier`：持枪移动速度倍率

修改后重启服务器即可生效。

## 地图

- 开阔训练场：远距离交火多，适合狙击和拉扯。
- 巷战街区：狭窄通道和拐角多，适合冲锋枪和霰弹枪。
- 中心竞技场：中心掩体争夺激烈，中距离武器较均衡。

切换地图会清空当前子弹、重开一局，并让所有玩家重新出生，避免卡进新地图障碍物。

## 道具

道具系统可以在游戏内左下角开启/关闭。开启后，服务器会在每张地图的固定刷新点随机生成道具，玩家碰到后自动拾取。

当前道具：

- 小治疗包：恢复生命值。
- 弹药箱：补满当前武器弹夹。
- 轻量化鞋：短时间提高移动速度。
- 护盾电池：获得临时护盾，受到伤害时先扣护盾。

道具配置放在：

```text
config/items.json
```

常用字段：

- `amount`：治疗量或护盾量
- `multiplier`：速度倍率
- `durationSeconds`：持续时间
- `spawnCooldownSeconds`：刷新点冷却
- `radius`：道具拾取半径和绘制半径
- `color`：道具颜色
